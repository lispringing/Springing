## 上位机接收部分
以下是运行了单帧检测脚本的效果
![截图](./img/udp.jpg)

FPGA发送的图像是960x540的图像，RGB565格式 这样的话是960x540x2=1036800

在这里要注意：按照FPGA端的发包逻辑 会发两种Udp数据包

一种是 972的（4包序列号＋4帧头代码＋4分辨率＋960数据流） 另一种是964的（4包序列号＋960数据流）

正常情况下1个972大小头帧包＋1079个964数据包可以组成一帧画面

接收脚本逻辑:开始监听udp，进行判断，如果是964的包全部丢弃，如果是972包，进行包帧头验证，若验证成功，开始生成一帧图像，直到再遇到另一个972包，依次循环

>注意，仓库提供了两种接收脚本，一种是python版本的，一种是c语言版本的，因为RK3568的cpu性能太弱了，我强烈推荐使用c语言版本，编译成机器码运行，可以大幅度节约资源占用

注意事项：

>在RK3568的Linux系统下，Linux内核给UDP缓冲层分配内存只有16MB，对于本项目是完全不够用的，需要用命令手动分配udp缓冲层

操作流程(在这里运行的是main.c)

打开本地网口1（看情况而定，如果你想开网口0也可以）
```
sudo ip link set eth1 up
```
设置网口1的ip地址为192.168.1.102
```
sudo ip addr add 192.168.1.102/24 dev eth1
```
验证ip是否修改成功
```
ip addr
```
修改Linux内核分配的缓冲层
```
# 接收缓冲区
sudo sysctl -w net.core.rmem_max=1073741824
sudo sysctl -w net.core.rmem_default=524288000

# 发送缓冲区
sudo sysctl -w net.core.wmem_max=1073741824
sudo sysctl -w net.core.wmem_default=524288000
```
编译c语言(main.c)
```
gcc main.c -o main -lpthread -lrt
```
运行脚本
```
sudo ./main
```

## OCR车牌识别

>基于 PyTorch 的中文车牌检测 + 识别项目，采用 CRNN 架构实现端到端车牌识别，可灵活切换模型大小、导出 ONNX 部署。

```
框架：PyTorch (>1.2.0)
核心算法：CRNN+CTC Loss
依赖：yaml、easydict、tensorboardX
数据源：CCPD+CRPD 公开数据集
```

经过训练后得到bset.pth 之后再转换为best.onnx

如何使用

安装python依赖项
```
pip install onnxsim -i https://pypi.tuna.tsinghua.edu.cn/simple
```
将训练的模型转换为onnx格式(去除部分特征，这样到时候转换成rknn后更适合rk3568运行)
```
python export.py --weights saved_model/best.pth --save_path saved_model/best.onnx  --simplify
```
使用onnx格式文件进行推理
```
python onnx_infer.py --onnx_file saved_model/best.onnx  --image_path images/test.jpg
```
>注意本项目是使用了onnx在cpu上进行推理(后面会提到为什么这样做)，所以文件夹中并没有best.rknn 如果需要可以进一步自行转换
